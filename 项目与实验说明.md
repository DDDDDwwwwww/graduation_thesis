# 项目与实验说明

## 1. 项目内容
这个项目在做的是：**面向通用游戏（GGP）的智能体系统**，比较“纯搜索方法”和“神经网络+搜索方法”在不同游戏上的表现，并分析它们在时间预算、搜索预算、数据规模、模型结构变化下的性能差异。

## 2. 实现的核心内容
项目实现了完整流水线：
- 规则解析与状态机：`gdl_parser.py`、`ggp_statemachine.py`
- 对局执行器：`game_runner.py`
- 智能体：
  - `RandomAgent`
  - `PureMCTAgent`（纯 MCTS 基线）
  - `HeuristicMCTSAgent`（带启发式/历史统计的 MCTS）
  - `ValueGreedyAgent`（只用价值网络选动作）
  - `NeuralValueMCTSAgent`（价值网络作为 MCTS 叶子评估）
- 训练与推理：`src/nn/*`
- 系统化实验脚本：`experiments/run_experiment_*.py`

## 3. 实验部分

## 3.1 A：基线强度对比
实验设置：3 个游戏：ticTacToe / connectFour / breakthrough、每组 10 局、`playclock=0.7`、`iterations=120`、CUDA。
- `pure_mct vs random`
- `heuristic_mcts vs pure_mct`
- `value_greedy:token_transformer vs random`
- `neural_mcts:token_transformer vs pure_mct`
- `neural_mcts:token_transformer vs heuristic_mcts`

## 3.2 B：时间预算敏感性（0.2s / 0.7s / 1.5s）
实验设置：时间预算 `[0.2, 0.7, 1.5]` 秒，`iterations=180`。
- 智能体：
  - `PureMCT`
  - `HeuristicMCTS`
  - `NeuralValueMCTS`
- 对战agent: `random`

## 3.3 C：搜索预算敏感性（50 / 120 / 300 次迭代）
实验设置：迭代数 `[50, 120, 300]`，`playclock=0.7`。
- 智能体：
  - `PureMCT`
  - `HeuristicMCTS`
  - `NeuralValueMCTS`
- 对战agent: `random`

## 3.4 D：数据规模敏感性（200 / 500 / 1000 / 2000 局数据）
实验设置：`connectFour`，数据规模 `[200, 500, 1000, 2000]`，每组 10 局。
- 智能体：
  - `value_greedy:fact_mlp`
  - `value_greedy:token_transformer`
  - `neural_mcts:fact_mlp`
  - `neural_mcts:token_mlp`
  - `neural_mcts:token_transformer`
- 对战agent: `random`

## 3.5 E：编码器/模型消融（fact_mlp、token_mlp、token_transformer）
实验设置：3 游戏、每组 10 局、`playclock=0.7`、`iterations=120`。
比较`NeuralValueMCTSAgent`以下三种核心配置：
  - `FactVectorEncoder + MLPValueNet`
  - `BoardTokenEncoder + MLPValueNet`
  - `BoardTokenEncoder + TransformerValueNet`

## 3.6 H：多游戏综合基准（ticTacToe / connectFour / breakthrough）
实验设置：3 游戏、每组 10 局、`playclock=0.7`、`iterations=120`。
- `pure_mct vs random`
- `neural_mcts:token_transformer vs pure_mct`



## 4. 关键结果

## 4.1 总体结论
- **纯搜索基线依然很强**：`pure_mct` 在多个游戏上都稳定。
- **神经 MCTS 不是“全面碾压”**：在部分游戏（如 breakthrough）有明显优势，但在 ticTacToe / connectFour 上不稳定。
- **预算增加有效，但有边际递减**：时间和迭代增加后胜率上升，但到一定程度收益变小。
- **数据量对 token_transformer 很关键**：小数据时表现弱，数据增大后明显改善。

## 4.2 代表性数字
1. A（基线强度）
跨游戏均值核心结果：
- `pure_mct vs random`：胜率 **0.867**，均分 86.7。
- `heuristic_mcts vs pure_mct`：胜率 **0.333**，平局率 0.200。
- `value_greedy:token_transformer vs random`：胜率 **0.400**。
- `neural_mcts:token_transformer vs pure_mct`：胜率 **0.500**（接近五五开）。
- `neural_mcts:token_transformer vs heuristic_mcts`：胜率 **0.367**。

2. B（时间预算）
对 random 的跨游戏胜率变化：
- `pure_mct`：0.633 -> 0.800 -> **0.867**
- `heuristic_mcts`：0.600 -> 0.800 -> **0.900**
- `neural_mcts:token_transformer`：0.800 -> 0.833 -> **0.933**
其中：
- `neural_mcts:token_transformer vs random`：
  - 0.2s: **0.800**
  - 0.7s: **0.833**
  - 1.5s: **0.933**
  时间预算提升能稳定`neural_mcts:token_transformer`增益。

3. C（搜索迭代）
对 random 的跨游戏胜率变化：
- `pure_mct`：0.833 -> 0.833 -> 0.800
- `heuristic_mcts`：0.833 -> 0.833 -> 0.833
- `neural_mcts:token_transformer`：0.733 -> 0.833 -> 0.833
`neural_mcts:token_transformer` 50→120 有明显提升，120→300 收益基本趋于平缓。

4. D（数据规模，connectFour）
对 random 的胜率变化（`cross_game.json`）：
- `value_greedy:fact_mlp`：0.70 -> 0.70 -> 0.70 -> 0.80
- `value_greedy:token_transformer`：0.30 -> 0.50 -> 0.70 -> 0.80
- `neural_mcts:fact_mlp`：1.00 -> 1.00 -> 1.00 -> 1.00
- `neural_mcts:token_mlp`：0.90 -> 1.00 -> 0.90 -> 0.80
- `neural_mcts:token_transformer`：0.70 -> 1.00 -> 1.00 -> 1.00
其中：
- `value_greedy:token_transformer vs random`：
  - 200: **0.30**
  - 500: **0.50**
  - 1000: **0.70**
  - 2000: **0.80**
- `neural_mcts:token_transformer vs random`：
  - 200: **0.70**
  - 500/1000/2000: **1.00**
数据规模扩大对 token_transformer 效果提升明显。fact_mlp 作为价值网络比较稳健。

5. E（编码器消融）
- value_greedy 对 random：`fact_mlp` 最好（**0.567**）
- neural_mcts 对 pure_mct：`token_transformer` 略好（**0.500**）
- 但 `token_transformer` 推理更慢（在 neural_mcts 下每步决策时间更高，约 0.285~0.291s/步）。

6. H（多游戏分项）
按游戏看（`by_game.json`）：
- `ticTacToe`
  - `pure_mct vs random`：0.8
  - `neural_mcts:token_transformer vs pure_mct`：0.0（平局 0.4）
- `connectFour`
  - `pure_mct vs random`：0.9
  - `neural_mcts:token_transformer vs pure_mct`：0.3
- `breakthrough`
  - `pure_mct vs random`：0.6
  - `neural_mcts:token_transformer vs pure_mct`：0.7
其中：
- `ticTacToe`：`neural_mcts:token_transformer vs pure_mct` 胜率 **0.0**
- `connectFour`：同对阵胜率 **0.3**
- `breakthrough`：同对阵胜率 **0.7**
神经方法在更复杂博弈上潜力更明显，但简单的游戏表现很差。
