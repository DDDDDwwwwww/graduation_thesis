# 第四章 系统设计与实现

建议字数：约 1800 字。

本章主要体现工程落地与实现链路，篇幅应控制在“讲清架构与关键实现”即可，不需要把代码逐行解释，但要让读者相信系统是真正可运行、可复现的。

本章用于从工程角度说明你的研究是如何真正落地的，强调方法不是停留在概念层，而是已经形成可运行、可训练、可评测的完整系统。

本章建议重点阐明以下内容：

- 系统整体架构，包括规则解析层、状态机层、搜索层、编码与神经网络层、实验编排层之间的关系。
- `games/`、`prolog/`、`src/`、`experiments/`、`outputs/` 等目录分别承担什么职责。
- GDL/KIF 规则解析与 Prolog 推理执行的实现流程，说明通用游戏状态机如何支持多种游戏。
- MCTS 核心模块与不同 agent 的实现思路，包括 `PureMCTAgent`、`HeuristicMCTSAgent`、`NeuralValueMCTSAgent` 的职责划分。
- 数据生成、数据转换、模型训练、模型加载、在线对局评测等关键工程流程。
- 实验结果如何被归档到 `meta / raw / summary / metrics / models / datasets / artifacts` 等结构中。
- 系统实现中为保证复现实验和对比公平性所采用的关键设计。
